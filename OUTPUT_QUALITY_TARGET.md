# Output Quality Target

This is not a generic code exercise.

The generated landing page should feel good enough to show to a founder for review.

It does not need to be perfect or production-final, but it should not feel like AI slop.

## Minimum Quality Bar

The generated output should:

- render as valid HTML in the preview
- look like a landing page, not raw notes or placeholder JSON
- have a clear hero section
- have at least one obvious CTA
- have readable spacing and section structure
- use the selected offer context coherently
- include the user’s notes in a meaningful way

## Stronger Quality Bar

If you have the core path working, the page should also feel:

- intentionally structured rather than auto-dumped
- specific to the offer rather than generic business copy
- plausible enough that a founder could react to the page itself, not just the code
- clean enough that export or copy would feel reasonable

## What To Avoid

- lorem ipsum
- broken or empty preview output
- markdown pretending to be HTML
- huge walls of unstructured text
- visually chaotic output that no founder would want to review

## Important Note

You are allowed to keep the scope narrow.

The goal is not to build a full design system.

The goal is to make one generated landing-page slice feel intentionally usable.
